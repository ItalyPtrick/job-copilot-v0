## 学习内容（~1 小时）

阅读 Doc 01 §1 的以下部分：

1. **为什么需要数据库？** — 理解当前项目纯内存运行的问题
2. **ORM 是什么？** — 理解 ORM 把数据库表映射为 Python 类的概念
3. **为什么选 SQLAlchemy 2.0** — 理解 `DeclarativeBase`、`Column` 类型映射、`sessionmaker`

重点理解：
- `engine` = 数据库连接引擎
- `SessionLocal` = 会话工厂，每次操作通过 session
- `Base` = 所有模型的基类
- `get_db()` = FastAPI 依赖注入，自动管理 session 生命周期

## 编码任务

### 任务 1：安装依赖（Doc 01 Step 1）

```bash
pip install sqlalchemy[asyncio] alembic aiosqlite redis
```

更新 `requirements.txt`，添加：
```
sqlalchemy[asyncio]>=2.0
alembic>=1.13
aiosqlite>=0.20
redis>=5.0
```

**验证**：`pip list | findstr sqlalchemy` 能看到版本号

### 任务 2：创建数据库连接配置（Doc 01 Step 2）

创建文件结构：
```
app/database/
├── __init__.py          # 导出 engine, SessionLocal, Base, get_db
└── connection.py        # 核心连接配置
```

`connection.py` 核心代码参考 Doc 01 Step 2：
- `DATABASE_URL` 从环境变量读取，默认 `sqlite:///./job_copilot.db`
- `engine = create_engine(DATABASE_URL, echo=False)`
- `SessionLocal = sessionmaker(bind=engine)`
- `class Base(DeclarativeBase): pass`
- `def get_db():` 使用 `yield` 模式
