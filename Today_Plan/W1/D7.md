## 学习内容（~1 小时）

### 复习 Doc 01 §6 面试要点

练习回答以下 4 个面试问题（Doc 01 §6）：

**Q1: 你为什么选 SQLAlchemy 而不是 Django ORM 或 Tortoise ORM？**
> 要点：事实标准、社区最大、FastAPI 集成方案成熟、2.0 原生 async

**Q2: 为什么先用 SQLite 后切 PostgreSQL？**
> 要点：渐进式架构、零配置快速迭代、SQLAlchemy 抽象层让切换只改连接字符串

**Q3: Redis 在你的项目中具体做了什么？**
> 要点：(1) 面试会话缓存 + TTL 自动过期 (2) Celery broker

**Q4: Alembic 和 JPA 的 ddl-auto 有什么区别？**
> 要点：显式迁移脚本、可审查、可回滚，比自动 ALTER TABLE 更可控

### 能讲出的亮点
- 渐进式架构：SQLite → PostgreSQL
- 依赖注入：FastAPI `Depends(get_db)`
- 迁移管理：Alembic 版本化 schema
- 缓存策略：Redis TTL 自动过期

### 任务 2：补充 W2-W4 预留模型骨架

在 `app/database/models/` 下创建 3 个文件（只写类框架，不实现字段）：

### 任务 3：端到端验证

按以下步骤完整走一遍：

1. 启动服务：`uvicorn app.main:app --reload`
2. 调用 `/task` 接口（用 curl 或 Postman）
3. 检查数据库：`sqlite3 job_copilot.db "SELECT id, task_type, status, created_at FROM task_records"`
4. 检查 Redis：`redis-cli ping`
5. 运行全部测试：`pytest tests/test_database.py tests/test_redis.py -v`
