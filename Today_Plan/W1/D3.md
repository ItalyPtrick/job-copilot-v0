## 学习内容（~1 小时）

阅读 Doc 01 §1 "数据库迁移是什么？"：

**核心概念**：
- 迁移 = 数据库的"版本控制"
- `alembic revision --autogenerate` = 对比模型和数据库，生成变更脚本
- `alembic upgrade head` = 执行迁移到最新版本
- `alembic downgrade -1` = 回滚一步
- 对标 Java 的 JPA `ddl-auto: update`，但 Alembic 更精确可控

**Alembic 文件结构**：
```
alembic/
├── versions/           # 迁移脚本存放处
├── env.py              # 配置文件（需修改）
└── script.py.mako      # 模板
alembic.ini              # 数据库连接配置（需修改）
```

## 编码任务

### 任务 1：初始化 Alembic（Doc 01 Step 4）

```bash
alembic init alembic
```

### 任务 2：修改 `alembic/env.py`

让 Alembic 能发现 SQLAlchemy 模型：

```python
# 在 env.py 中添加
from app.database.connection import Base
from app.database.models import *  # 导入所有模型

target_metadata = Base.metadata
```

### 任务 3：修改 `alembic.ini`

设置数据库连接：
```ini
sqlalchemy.url = sqlite:///./job_copilot.db
```

### 任务 4：生成并执行首次迁移

```bash
# 生成迁移脚本
alembic revision --autogenerate -m "initial tables"

# 执行迁移
alembic upgrade head
```

### 任务 5：验证迁移三连

```bash
alembic upgrade head
alembic downgrade base
alembic upgrade head
```

## 验收标准

- [x] `alembic/versions/` 目录下有一个迁移脚本文件
- [x] `alembic upgrade head` / `downgrade base` / `upgrade head` 三连成功无报错
- [x] `job_copilot.db` 文件存在于项目根目录
- [x] `sqlite3 job_copilot.db ".tables"` 输出包含 `task_records` 和 `alembic_version`
- [x] `sqlite3 job_copilot.db ".schema task_records"` 能看到完整建表语句
