## 学习内容（~1 小时）

### 为什么用内存 SQLite 做测试？

- `sqlite:///:memory:` 创建内存数据库，测试结束自动销毁
- 每个测试用例独立的数据库，互不干扰
- 不会污染开发环境的 `job_copilot.db`

### pytest fixture 模式

```python
@pytest.fixture
def db_session():
    engine = create_engine("sqlite:///:memory:")
    Base.metadata.create_all(engine)
    Session = sessionmaker(bind=engine)
    session = Session()
    yield session        # 测试用例在这里执行
    session.close()      # 测试结束后清理
```

关键点：
- `yield` 之前 = setup（创建数据库 + 表）
- `yield` 的值 = 注入到测试函数的参数
- `yield` 之后 = teardown（关闭 session）
- 每个测试函数都会重新执行 fixture

## 编码任务

### 创建 `tests/test_database.py`

参考 Doc 01 §5 的测试代码，实现以下用例：

#### 用例 1：`test_create_task_record`
- 创建一条 TaskRecord，commit 后查询
- 断言 `task_type`、`status` 与创建时一致
- 断言 `id` 自动生成（非 None）

#### 用例 2：`test_query_by_task_type`
- 插入 3 条记录：2 条 `jd_analyze`，1 条 `resume_optimize`
- 按 `task_type == "jd_analyze"` 过滤查询
- 断言返回 2 条记录

#### 用例 3：`test_task_record_json_fields`
- 创建记录时 `payload={"jd_text": "test", "list": [1, 2, 3]}`
- 查询后断言 `payload["jd_text"] == "test"`
- 断言 `payload["list"] == [1, 2, 3]`
- 验证 JSON 字段的存取完整性

#### 用例 4（可选加分）：`test_created_at_auto_set`
- 创建记录时不传 `created_at`
- 查询后断言 `created_at is not None`
