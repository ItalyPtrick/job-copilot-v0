## 学习内容（~1.5 小时）

### Redis 在本项目中的角色

阅读 Doc 01 §1 "为什么需要 Redis？"：

| 角色 | 用途 | 对标 |
|---|---|---|
| **会话缓存** | 模拟面试多轮对话上下文 | 替代 ConcurrentHashMap |
| **任务队列 Broker** | Celery 的消息中间件 | Redis Stream |

### 为什么不用内存 dict？
- 服务重启后丢失
- 多 worker 进程无法共享
- Redis 支持 TTL 自动过期

### 关键概念
- **TTL（Time To Live）**：key 的自动过期时间（秒）
- **`setex(key, ttl, value)`**：设置带过期时间的 key
- **`json.dumps` / `json.loads`**：Redis 存字符串，Python dict 需序列化

### 本地安装 Redis

Windows 推荐方案：
1. **方案 A**：WSL2 中 `sudo apt install redis-server && sudo service redis-server start`
2. **方案 B**：Docker `docker run -d --name redis -p 6379:6379 redis:7`
3. **方案 C**：Memurai（Windows 原生 Redis 兼容）

验证：`redis-cli ping` → 返回 `PONG`

## 编码任务

### 任务 1：创建 Redis 客户端封装（Doc 01 Step 6）

创建文件结构：
```
app/cache/
├── __init__.py
└── redis_client.py
```

`redis_client.py` 参考 Doc 01 Step 6，实现 3 个函数：

| 函数 | 功能 | 参数 |
|---|---|---|
| `set_session(session_id, data, ttl=3600)` | 缓存面试会话 | session_id: str, data: dict |
| `get_session(session_id)` | 获取面试会话 | 返回 dict 或 None |
| `delete_session(session_id)` | 删除面试会话 | — |

关键实现：
- `redis_client = redis.from_url(REDIS_URL, decode_responses=True)`
- `REDIS_URL` 从环境变量读取，默认 `redis://localhost:6379/0`
- 存储时 `json.dumps(data, ensure_ascii=False)`
- 读取时 `json.loads(raw)`

### 任务 2：编写 Redis 测试

创建 `tests/test_redis.py`：

#### 用例 1：`test_set_and_get_session`（Mock Redis）
- 使用 `unittest.mock.patch` Mock `redis_client`
- 验证 `set_session` 调用了 `setex`
- 验证 `get_session` 返回正确的 dict

#### 用例 2：`test_delete_session`（Mock Redis）
- 验证 `delete_session` 调用了 `redis_client.delete`

#### 用例 3（可选）：`test_redis_smoke`（真实连接）
- 用 `pytest.mark.skipif` 判断 Redis 是否可用
- 真实写入、读取、删除，验证完整流程
