## 学习内容（~2 小时）

### Part 1：面试复习（Doc 02 §6）

练习回答以下 5 个面试问题：

**Q1: 解释一下你的 RAG 系统的完整流程？**
> 要点：文档上传 → PyMuPDF/python-docx 解析 → RecursiveCharacterTextSplitter 分块（500字/100重叠）→ OpenAI Embedding 向量化（1536维）→ ChromaDB 存储 → 查询时向量化问题 → top-k 相似度搜索 → 拼入 prompt → LLM 生成 → SSE 流式返回 + 来源引用

**Q2: 你的分块策略怎么选的？chunk_size 和 overlap 怎么定？**
> 要点：chunk_size=500 经验值平衡（太小丢上下文，太大引入噪声）；overlap=100 保证边界信息不截断；RecursiveCharacterTextSplitter 优先按段落/句子切割；实际可通过评估检索准确率调优

**Q3: ChromaDB 和 pgvector 的区别？你为什么先用 ChromaDB？**
> 要点：ChromaDB 嵌入式零依赖适合开发；pgvector 是 PG 扩展可和业务数据同库；先 ChromaDB 因为开发效率高，后续切 PG 后迁移到 pgvector

**Q4: RAG 的检索质量如何保证？有什么优化手段？**
> 要点：(1) 分块策略优化 (2) 混合检索（BM25+向量） (3) Re-ranking（交叉编码器重排） (4) Query Expansion（LLM 改写问题）

**Q5: RAG 和 Fine-tuning 的区别，什么时候该用哪个？**
> 要点：RAG 访问外部知识，Fine-tuning 改变行为/风格；RAG 实时更新、成本低、可解释；Fine-tuning 适合深度领域理解

### 能讲出的亮点
- 完整 RAG 流水线：上传→解析→分块→向量化→检索→生成
- 流式响应：SSE 打字机效果
- 与现有架构集成：RetrieverContext 注入到 Orchestrator
- 渐进式存储：ChromaDB → pgvector
- 多格式支持：PDF/DOCX/MD/TXT

### Part 2：进阶探索（~1 小时，只看概念不写代码）

了解以下 3 个进阶概念，为后续优化和面试准备：

#### 混合检索（Hybrid Search）
- **思路**：向量检索（找语义相关）+ BM25 关键词检索（找精确匹配），两者互补
- **实现**：LangChain 的 `EnsembleRetriever` 可以组合多个 retriever
- **RRF（Reciprocal Rank Fusion）**：融合两种检索结果的排序算法
- **面试怎么讲**："纯向量检索可能漏掉精确关键词（如 API 名称），BM25 能补充精确匹配；两者用 RRF 融合后效果最好"

#### Reranking（重排序）
- **思路**：先用向量检索粗排 top-20，再用 Cross-Encoder 精排出 top-5
- **Cross-Encoder vs Bi-Encoder**：Bi-Encoder 分别编码 query 和 doc 再算相似度（快但精度低）；Cross-Encoder 同时编码 query+doc 对（慢但精度高）
- **面试怎么讲**："检索分两阶段——召回阶段用 Bi-Encoder 快速找候选，精排阶段用 Cross-Encoder 重排提升精度"

#### RAG 评估（Evaluation）
- **常见指标**：Faithfulness（回答是否忠实于文档）、Answer Relevancy（回答是否相关）、Context Precision（检索的文档是否相关）
- **Ragas 框架**：Python RAG 评估框架，可自动计算上述指标
- **面试怎么讲**：“我调研过 Ragas 评估框架，主要关注 faithfulness 和 answer_relevancy 两个指标，计划在优化阶段集成进来”

> 说“调研过/计划集成”而不是“我用了”——你还没实际实现，面试说假话会被追问穿。

> 这 3 个概念现在只需理解原理和面试怎么讲。后续有时间再实现代码。

## 编码任务

### 任务 1：用自己的文档测试 RAG

启动服务后，上传你的 `docs/` 目录中的文档来测试：

```bash

uvicorn app.main:app --reload

# 上传 02-rag-knowledge-base.md
curl -X POST http://localhost:8000/kb/upload -F "file=@docs/02-rag-knowledge-base.md" -F "collection_name=project_docs"
```
