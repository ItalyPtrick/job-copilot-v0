## 学习内容（~1.5 小时）

阅读 Doc 02 §1 的前 4 个小节：

### 1. 什么是 RAG？
- 理解 RAG 的完整流程：用户问题 → 检索文档片段 → 拼入 prompt → LLM 生成回答
- 理解为什么需要 RAG：解决幻觉、知识更新、私有数据、成本低于 Fine-tuning

### 2. RAG vs Fine-tuning
- RAG 适合"让 LLM 访问外部知识"，Fine-tuning 适合"改变 LLM 行为/风格"
- 记住表格中的 4 个对比维度（适用场景、数据更新、成本、可解释性）——面试常问

### 3. Embedding（向量嵌入）
- 理解文本 → 高维向量的转换，语义相似的文本向量距离更近
- 了解 4 个常用模型的区别（OpenAI / bge / MiniLM）
- **Doc 02 示例使用 `text-embedding-3-small`（1536 维）**，用于理解 embedding 接入方式

### 4. 文档分块策略
- 理解 4 种分块策略：固定长度 / 递归字符 / 语义 / Markdown Header
- **后续实现计划使用 `RecursiveCharacterTextSplitter`**，chunk_size=500, overlap=100
- 理解 overlap 的作用：保证分块边界处的信息不被截断

重点理解：
- **RAG 三步走**：索引（文档→分块→向量化→存储）→ 检索（查询→相似度匹配）→ 生成（拼 prompt→LLM 回答）
- **chunk_size 和 overlap 的权衡**：太小丢上下文，太大引入噪声

### 任务 3：创建 ChromaDB 封装（Doc 02 Step 1）

创建 `app/modules/knowledge_base/vector_store.py`，参考 Doc 02 Step 1 的代码：

核心内容：
- `CHROMA_DIR = "./data/chroma"` — 向量数据库持久化目录
- `embeddings = OpenAIEmbeddings(...)` — Embedding 模型由 `OPENAI_EMBEDDING_MODEL` 驱动
- `get_vector_store(collection_name)` — 获取或创建向量存储集合
- `add_documents(collection_name, documents)` — 向集合中添加文档块
- `search(collection_name, query, top_k=5)` — 相似度检索

**注意**：
- 确保 `.env` 中 `OPENAI_EMBEDDING_API_KEY`、`OPENAI_EMBEDDING_BASE_URL`、`OPENAI_EMBEDDING_MODEL` 已配置
- 文件顶部加 `from dotenv import load_dotenv; load_dotenv()`，否则 `OpenAIEmbeddings` 读不到环境变量

**当前项目实际接入说明**：
- 当前项目已将聊天模型配置与 embedding 配置拆分。
- 若使用阿里云百炼兼容 embeddings 接口，`OpenAIEmbeddings` 需设置 `check_embedding_ctx_length=False`，以避免 LangChain 默认 token 预切分产生的 token 数组输入与兼容接口不匹配。
- 当前仓库实际接入的 embedding 模型为 `text-embedding-v4`。

### 任务 4：快速验证 ChromaDB

写一个临时脚本验证 ChromaDB 能工作：

```python
# 在项目根目录运行：
python -c "
from app.modules.knowledge_base.vector_store import get_vector_store, add_documents, search
from langchain.schema import Document

# 写入测试数据
docs = [Document(page_content='Python 是一种编程语言', metadata={'source': 'test'})]
add_documents('quick_test', docs)

# 检索
results = search('quick_test', '什么是 Python', top_k=1)
print(f'检索到 {len(results)} 条结果：{results[0].page_content}')
print('ChromaDB 验证通过！')
"
```
