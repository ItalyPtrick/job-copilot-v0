## 学习内容（~1 小时）

### LangChain LCEL（LangChain Expression Language）

LangChain 用 `|` 管道符组合多个组件，类似 Unix 管道：

```python
chain = prompt | llm | output_parser
result = chain.invoke({"key": "value"})
```

**核心组件**：
- **`ChatPromptTemplate`**：定义 prompt 模板，用 `{variable}` 占位
- **`ChatOpenAI`**：LLM 调用，支持 `streaming=True` 流式输出
- **`StrOutputParser`**：将 LLM 输出的 AIMessage 转为纯字符串

**流式调用**：
```python
# 同步调用
result = chain.invoke({"context": "...", "question": "..."})

# 异步流式（SSE 用）
async for chunk in chain.astream({"context": "...", "question": "..."}):
    yield chunk  # 每次返回一小段文字
```

重点理解：
- `chain.invoke()` = 同步调用，等全部生成完才返回
- `chain.astream()` = 异步流式，边生成边返回，用于 SSE
- 为什么用 `temperature=0`：RAG 场景需要忠实于文档内容，不需要创造性

## 编码任务

### 任务 1：创建 RAG 问答链（Doc 02 Step 3）

创建 `app/modules/knowledge_base/rag_chain.py`，参考 Doc 02 Step 3 的代码：

核心内容：

**RAG Prompt 模板**：
```
你是一个知识库问答助手。根据以下检索到的文档内容回答用户问题。
如果文档中没有相关信息，请明确说明"根据现有知识库内容，我无法回答这个问题"。

检索到的文档内容：
{context}

用户问题：{question}

请基于文档内容回答：
```

**两个函数**：

| 函数 | 功能 | 返回值 |
|---|---|---|
| `rag_query(collection_name, question, top_k=5)` | 非流式问答 | `{"answer": str, "sources": list}` |
| `rag_query_stream(collection_name, question, top_k=5)` | 流式问答（async generator） | 每次 yield 一个文本片段（不返回 `sources`） |

**实现要点**：
1. 先调用 `search(collection_name, question, top_k)` 检索文档块
2. 将检索到的文档内容用 `"\n\n---\n\n"` 拼接为 context
3. 非流式：`chain.invoke()`；流式：`chain.astream()`
4. sources 中包含每个文档块的 content（截取前 50 字）、source_file、chunk_index
5. 流式版本当前只负责输出回答文本，来源信息暂不进入流

**注意**：
- `ChatOpenAI` 初始化时用 `model` 参数，注意从 `.env` 读取 `OPENAI_MODEL`
- `streaming=True` 要加上，即使非流式调用也不影响，但流式调用必须有

### 任务 3：验证流式问答

```python
python -c "
import asyncio
from app.modules.knowledge_base.rag_chain import rag_query_stream

async def test():
    chunks = []
    async for chunk in rag_query_stream('quick_test', '什么是 Python'):
        chunks.append(chunk)
        print(chunk, end='', flush=True)
    print(f'\n\n总共 {len(chunks)} 个 chunk')
    print('流式问答验证通过！')

asyncio.run(test())
"
```
