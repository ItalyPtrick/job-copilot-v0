## 学习内容（~1 小时）

阅读 Doc 02 §1 的后 2 个小节：

### 1. 向量数据库对比
- **ChromaDB**：嵌入式、零配置、Python 原生 → 本项目初期选择
- **pgvector**：PostgreSQL 扩展、复用 PG、支持 SQL 过滤 → Phase 3 迁移目标
- **FAISS**：内存库、速度最快但不持久化 → 大规模检索场景
- **Milvus**：独立服务、功能最全但部署复杂 → 企业级

面试关键：**能讲清楚为什么先 ChromaDB 后 pgvector**（渐进式架构，开发效率 → 生产稳定性）

### 2. 检索策略
- **相似度搜索**：返回与查询向量最近的 top-k → 本项目先用
- **MMR（最大边际相关）**：在相关性和多样性之间平衡 → 后续可加
- **混合检索**：向量 + BM25 关键词 → D7 进阶探索主题

重点理解：
- 为什么纯向量检索可能不够：同义词能找到，但精确关键词（如 API 名称）可能漏掉
- 混合检索的思路：向量找语义相关 + BM25 找关键词精确匹配，两者互补

## 编码任务

### 任务 1：创建文档加载器（Doc 02 Step 2）

创建 `app/modules/knowledge_base/document_loader.py`，参考 Doc 02 Step 2 的代码：

核心内容：

**格式映射字典**：
| 扩展名 | Loader 类 | 说明 |
|---|---|---|
| `.pdf` | `PyMuPDFLoader` | PDF 解析（pymupdf） |
| `.docx` | `Docx2txtLoader` | Word 解析（python-docx） |
| `.txt` | `TextLoader` | 纯文本 |
| `.md` | `UnstructuredMarkdownLoader` | Markdown |

**文本分块器**：
```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=100,
    separators=["\n\n", "\n", "。", "！", "？", ".", " ", ""],
)
```

**`load_and_split(file_path)` 函数**：
1. 根据文件扩展名选择对应的 Loader
2. `loader.load()` 加载文档
3. `text_splitter.split_documents(documents)` 分块
4. 为每个 chunk 添加 `chunk_index` 和 `source_file` 元数据
5. 返回 chunks 列表

**注意**：记得在文件顶部 `from pathlib import Path`。
