## 学习内容（~1 小时）

### FastAPI 文件上传

```python
from fastapi import UploadFile, File, Form

@router.post("/upload")
async def upload(
    file: UploadFile = File(...),            # 文件参数
    collection_name: str = Form(default="default"),  # 表单字段
):
    # file.filename — 原始文件名
    # file.file — 类文件对象，用 shutil.copyfileobj 保存
    pass
```

重点：
- `File(...)` 和 `Form(...)` 不能和 JSON body 混用
- 上传大文件时用 `shutil.copyfileobj` 流式写入，不要 `file.read()` 全部读入内存

### SSE（Server-Sent Events）

SSE 是服务器向客户端单向推送的协议，比 WebSocket 简单：

```python
from sse_starlette.sse import EventSourceResponse

async def event_generator():
    yield {"event": "message", "data": "一段文字"}
    yield {"event": "done", "data": "[DONE]"}

return EventSourceResponse(event_generator())
```

重点：
- 每个 event 包含 `event`（类型）和 `data`（内容）
- 客户端通过 `EventSource` 或 `fetch` 消费
- 流结束时发送 `[DONE]` 信号

## 编码任务

### 任务 1：创建知识库路由（Doc 02 Step 4）

创建 `app/modules/knowledge_base/router.py`，参考 Doc 02 Step 4 的代码：

**4 个路由**：

| 方法 | 路径 | 功能 | 参数 |
|---|---|---|---|
| `POST` | `/kb/upload` | 上传文档并向量化 | `file: UploadFile`, `collection_name: str` |
| `POST` | `/kb/query` | 非流式知识库问答 | `question: str`, `collection_name: str` |
| `POST` | `/kb/query/stream` | SSE 流式问答 | `question: str`, `collection_name: str` |
| `GET` | `/kb/collections` | 列出所有集合 | 无 |

**`/kb/upload` 实现要点**：
1. 生成 `file_id = str(uuid.uuid4())`
2. 保存文件到 `./data/uploads/{file_id}{ext}`
3. 调用 `load_and_split(save_path)` 加载并分块
4. 调用 `add_documents(collection_name, chunks)` 向量化
5. 返回 `{"status": "success", "file_id": ..., "filename": ..., "chunks_count": ...}`

**错误处理**：用 try/except 包裹 `load_and_split`，捕获 `ValueError`（不支持的文件格式）返回 400：
```python
from fastapi import HTTPException

try:
    chunks = load_and_split(save_path)
except ValueError as e:
    raise HTTPException(status_code=400, detail=str(e))
```

**`/kb/query` 实现要点**：
- 调用 `rag_query(collection_name, question)` 并返回结果

**`/kb/query/stream` 实现要点**：
- 定义 `async def event_generator()`
- 内部调用 `rag_query_stream(collection_name, question)`
- 每个 chunk 包装为 `{"event": "message", "data": chunk}`
- 结束时发送 `{"event": "done", "data": "[DONE]"}`
- 返回 `EventSourceResponse(event_generator())`
- 如果前端后续需要展示来源信息，在路由层扩展 SSE 事件结构；D3 的 `rag_chain` 先只负责流式输出回答文本

**`/kb/collections` 实现要点**：
- 用 `chromadb.PersistentClient(path="./data/chroma")` 获取客户端
- `client.list_collections()` 列出集合
- 返回每个集合的 name 和 count

**注意**：
- 记得 `os.makedirs("./data/uploads", exist_ok=True)` 确保上传目录存在
- `router = APIRouter(prefix="/kb", tags=["知识库"])`
- 记得从 `pathlib` 导入 `Path` 获取文件扩展名

### 任务 2：注册路由到主应用（Doc 02 Step 5）

修改 `app/main.py`，添加：

```python
from app.modules.knowledge_base.router import router as kb_router

app.include_router(kb_router)
```

> Step 5 只有 2 行代码，和路由创建放在同一天更顺畅。
