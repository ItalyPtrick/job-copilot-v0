## 学习内容（~1h）

### Celery 架构

```text
FastAPI（生产者）
    ↓ 发送任务消息
Redis（Broker，消息队列）
    ↓ Worker 从队列取任务
Celery Worker（消费者，执行任务）
    ↓ 执行完毕，存结果
Redis（Result Backend，结果存储）
```

核心概念：
- **Broker**：消息中间件（本项目用 Redis），负责传递任务消息
- **Worker**：独立进程，监听队列并执行任务
- **Result Backend**：存储任务执行结果，支持查询状态

### Windows 注意事项

Celery 4+ 不原生支持 Windows 多进程。解决方案：

```bash
# --pool=solo 表示单进程顺序执行，Windows 可用
celery -A celery_app worker --pool=solo --loglevel=info
```

### Task 重试机制

```python
@celery.task(bind=True, max_retries=3, default_retry_delay=10)
def my_task(self, ...):
    try:
        ...
    except Exception as e:
        raise self.retry(exc=e)  # 触发重试
```

- `bind=True`：task 方法接收 self 参数，可以调用 `self.retry()`
- `max_retries=3`：最多重试 3 次
- `default_retry_delay=10`：每次重试间隔 10 秒

**面试怎么讲**："Celery Worker 从 Redis 队列取任务执行，LLM 调用失败时通过 `self.retry()` 自动重试最多 3 次，间隔 10 秒。3 次都失败则标记任务为 FAILURE。"

## 编码任务

### 任务 1：安装依赖

```bash
pip install celery
```

更新 `requirements.txt` 添加 `celery>=5.4`。

### 任务 2：创建 celery_app.py

在项目根目录创建 `celery_app.py`：

```python
import os
from dotenv import load_dotenv
from celery import Celery

load_dotenv()  # Worker 独立进程启动，需要自己加载 .env
CELERY_BROKER_URL = os.getenv("CELERY_BROKER_URL", "redis://localhost:6379/1")

celery = Celery("job_copilot", broker=CELERY_BROKER_URL)
celery.conf.update(
    result_backend=CELERY_BROKER_URL,
    task_serializer="json",
    result_serializer="json",
    accept_content=["json"],
    timezone="Asia/Shanghai",
    task_track_started=True,
)
celery.autodiscover_tasks(["app.modules.resume"])
```

### 任务 3：更新 .env

添加一行：`CELERY_BROKER_URL=redis://localhost:6379/1`

> 使用 Redis DB 1，与主应用的 DB 0 隔离。

### 任务 4：创建 service.py

创建 `app/modules/resume/service.py`，数据库操作封装：

- `create_resume_record(db, resume_id, filename, file_path, file_ext, target_role) -> ResumeRecord`
- `get_resume_by_id(db, resume_id) -> ResumeRecord | None`
- `get_resume_by_hash(db, content_hash) -> ResumeRecord | None`
- `update_resume_status(db, resume_id, status, result=None, error=None)`

每个函数内部使用 SQLAlchemy session 查询/更新。

### 任务 5：创建 tasks.py

创建 `app/modules/resume/tasks.py`：

```python
@celery.task(bind=True, max_retries=3, default_retry_delay=10)
def analyze_resume_task(self, resume_id: str, file_path: str, target_role: str = ""):
    # 1. 解析文档 → raw_text
    # 2. 计算 content_hash
    # 3. 查数据库是否已有相同 (hash + target_role) → 有则复用结果
    # 4. 无则调用 LLM 分析
    # 5. 更新数据库状态（completed / failed）
    ...
```

关键点：
- task 内部需要自己创建 DB session（Celery Worker 是独立进程）
- 解析失败或 LLM 失败时更新状态为 "failed" + error_message
- 去重查询同时匹配 `content_hash` 和 `target_role`，命中则复用已有 result
- 并发上传同一份简历时，捕获 `IntegrityError` 返回已有记录（参考 W2-D5 的幂等处理）
