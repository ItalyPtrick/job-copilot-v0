## 学习内容（~1h）

阅读 Doc 04 §1 第一节「文档解析」：

### 核心问题

简历格式多样（PDF / DOCX / TXT），需要先提取纯文本才能送 LLM 分析。每种格式的解析逻辑不同：

| 格式 | 库 | 核心方法 |
|---|---|---|
| PDF | PyMuPDF (`fitz`) | 逐页 `page.get_text()` 拼接 |
| DOCX | python-docx | 遍历 `doc.paragraphs` 拼接 |
| DOC | python-docx | 复用 DOCX 解析器（大多数 .doc 兼容） |
| TXT | 内置 `open()` | 直接读取 |

### 设计模式

策略模式：定义一个 `PARSERS` 字典，将文件后缀映射到对应的解析函数。统一入口 `parse_resume(file_path)` 自动按后缀分派。

```python
PARSERS = {
    ".pdf": parse_pdf,
    ".docx": parse_docx,
    ".doc": parse_docx,  # 复用 DOCX 解析器
    ".txt": parse_txt,
}

def parse_resume(file_path: str) -> str:
    ext = Path(file_path).suffix.lower()
    parser = PARSERS.get(ext)
    if not parser:
        raise ValueError(f"不支持的文件格式: {ext}")
    return parser(file_path)
```

**面试怎么讲**："用策略模式将文件后缀映射到对应解析器，新增格式只需加一个函数 + 一行注册，符合开闭原则。"

### 任务 3：实现 parser.py

创建 `app/modules/resume/parser.py`，包含：

- `parse_pdf(file_path: str) -> str`：PyMuPDF 逐页提取文本
- `parse_docx(file_path: str) -> str`：python-docx 遍历段落（同时处理 .doc）
- `parse_txt(file_path: str) -> str`：UTF-8 读取
- `parse_resume(file_path: str) -> str`：统一入口，按后缀分派（PARSERS 包含 .pdf/.docx/.doc/.txt）

关键实现细节：
- PDF 解析后 `strip()` 去首尾空白
- DOCX 过滤空段落（`if p.text.strip()`）
- TXT 支持 UTF-8 编码
