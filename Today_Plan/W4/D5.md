## 学习内容（~30 分钟）

### FastAPI 文件上传

```python
from fastapi import UploadFile, File, Form

@router.post("/upload")
async def upload(file: UploadFile = File(...), target_role: str = Form(default="")):
    # file.filename → 原始文件名
    # file.file → 类文件对象，用 shutil.copyfileobj 保存
    ...
```

### FileResponse 返回文件

```python
from fastapi.responses import FileResponse

@router.get("/{resume_id}/export")
async def export_pdf(resume_id: str):
    return FileResponse(pdf_path, filename=f"report_{resume_id}.pdf", media_type="application/pdf")
```

### 异步状态轮询

用户上传后拿到 `resume_id`，然后通过 `GET /resume/{id}/status` 轮询直到状态变为 `completed`。比 WebSocket 简单，适合分析耗时可预期（5-30 秒）的场景。

**面试怎么讲**："上传后立即返回 HTTP 202 Accepted + resume_id，表示任务已接受但未完成。客户端轮询状态。选轮询而非 WebSocket 是因为分析通常 10 秒内完成，轮询实现简单、前端无需维护长连接。"

### 任务 2：创建 router.py

创建 `app/modules/resume/router.py`，包含 5 个端点：

| 端点 | 方法 | 功能 |
|---|---|---|
| `/resume/upload` | POST | 上传文件 + 触发异步分析 |
| `/resume/{resume_id}/status` | GET | 查询分析状态 |
| `/resume/{resume_id}/report` | GET | 获取结构化分析结果 |
| `/resume/{resume_id}/export` | GET | 下载 PDF 报告 |
| `/resume/list` | GET | 历史记录列表（分页） |

`POST /upload` 流程：

```python
async def upload_resume(file: UploadFile, target_role: str = Form(default="")):
    # 1. 生成 resume_id (uuid4)
    # 2. 保存文件到 data/resumes/{resume_id}{ext}
    # 3. 创建 DB 记录（status=pending）
    # 4. 触发 Celery task: analyze_resume_task.delay(resume_id, file_path, target_role)
    # 5. 返回 202 + {"resume_id": ..., "status": "analyzing"}
    # 注意：并发上传相同简历时，捕获 IntegrityError 返回已有记录（参考 W2-D5）
```

`GET /export` 流程：

```python
async def export_pdf(resume_id: str):
    # 1. 查数据库拿 analysis_result
    # 2. 检查状态是否 completed
    # 3. 生成 PDF（如果不存在）
    # 4. 返回 FileResponse
```

### 任务 4：基础 API 测试

创建 `tests/test_resume_api.py`，先写基础用例（D6 补充 edge case）：

```python
from fastapi.testclient import TestClient
from app.main import app

client = TestClient(app)

def test_upload_returns_resume_id():
    with open("tests/fixtures/test_resume.txt", "rb") as f:
        resp = client.post("/resume/upload", files={"file": ("resume.txt", f)})
    assert resp.status_code == 202
    assert "resume_id" in resp.json()

def test_upload_invalid_format():
    resp = client.post("/resume/upload", files={"file": ("bad.exe", b"data")})
    assert resp.status_code == 400
```

### 任务 5：端到端手动测试

需要两个终端：

```bash
# 终端 1：启动 FastAPI
uvicorn app.main:app --reload

# 终端 2：启动 Celery Worker
celery -A celery_app worker --pool=solo --loglevel=info
```

测试流程：

```bash
# 1. 上传
curl -X POST http://localhost:8000/resume/upload \
  -F "file=@tests/fixtures/test_resume.txt" \
  -F "target_role=Python开发"
# 期望返回 202 + {"resume_id": "xxx", "status": "analyzing"}

# 2. 查状态（等几秒）
curl http://localhost:8000/resume/xxx/status
# 期望返回 {"status": "completed"}

# 3. 获取报告
curl http://localhost:8000/resume/xxx/report
# 期望返回结构化 JSON

# 4. 导出 PDF
curl -o report.pdf http://localhost:8000/resume/xxx/export
# 期望下载 PDF 文件
```
