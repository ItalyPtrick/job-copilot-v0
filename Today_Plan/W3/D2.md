## 学习内容（~1 小时）

阅读 Doc 03 §1 的后 2 个小节：

### 1. Skill 驱动出题（Doc 03 §1 第 3 节）

- 理解 Skill 的设计思路：用 Markdown 文件定义每个面试方向的考察范围和难度分布
- Skill 文件结构：
  - **考察范围**：列出该方向的知识点（如 Python 基础、Web 框架、数据库、异步编程、系统设计）
  - **难度分布**：基础题 40% / 进阶题 40% / 开放题 20%
  - **参考知识库**：可选，关联 RAG 知识库的 collection
- **面试怎么讲**："面试方向通过 Markdown Skill 文件配置，可以灵活扩展新方向，不需要改代码"

### 2. 评估引擎（Doc 03 §1 第 4 节）

- 理解"分批评估 + 结构化输出 + 二次汇总 + 降级兜底"架构：

```
面试结束 → Step 1: 按题目分批评估（每 3 题一批）
         → Step 2: 汇总所有单题评分，生成总评报告
         → Step 3: 某一步失败则降级重试
```

- **为什么分批而非一次性**：一次评估 20 题 prompt 太长，LLM 注意力分散；分批 3 题一组更精准，还能并发加速
- **面试怎么讲**："分批评估保证质量，每批只处理 3 题让 LLM 更专注；分批还能并发调用加速；最后汇总生成总评报告"

重点理解：
- **Skill 文件 → prompt 模板 → LLM 出题** 的完整链路
- **评估引擎的分层架构**：单题评分 → 批次汇总 → 总评报告

## 编码任务

### 任务 1：创建 Skill 定义文件

创建 `app/skills/python_backend.md`，参考 Doc 03 §1 第 3 节的示例：

```markdown
# skill: python_backend.md

## 考察范围
- Python 基础：数据结构、装饰器、生成器、GIL
- Web 框架：FastAPI / Flask 路由、中间件、依赖注入
- 数据库：SQLAlchemy ORM、事务、连接池
- 异步编程：asyncio、协程、并发模式
- 系统设计：缓存策略、消息队列、微服务拆分

## 难度分布
- 基础题：40%
- 进阶题：40%
- 开放题：20%

## 参考知识库
- collection: python_docs
```

同时创建 `app/skills/` 目录（如果不存在）。

### 任务 2：创建 Session 管理（Doc 03 Step 2）

创建 `app/modules/interview/session_manager.py`，参考 Doc 03 Step 2 的代码：

核心内容：
- `SESSION_PREFIX = "interview:session:"` — Redis key 前缀，与现有 session 区分
- `SESSION_TTL = 7200` — 2 小时过期
- `create_session(config)` → 生成 UUID，存入 Redis，返回 session_id
- `get_session(session_id)` → 从 Redis 读取 session 数据
- `update_session(session_id, data)` → 更新 session 数据

**注意**：
- 直接复用 `app/cache/redis_client.py` 中的 `redis_client` 实例
- session 数据结构：`{"session_id", "config", "status", "messages", "questions_asked", "current_question_index"}`
- `messages` 是完整对话历史数组（全量历史策略），格式与 OpenAI messages 一致

### 任务 3：验证 Session CRUD

```bash
# 需要 Redis 运行中
python -c "
from app.modules.interview.session_manager import create_session, get_session, update_session

# 创建
sid = create_session({'skill': 'python_backend', 'total_questions': 5})
print(f'Created session: {sid}')

# 读取
session = get_session(sid)
print(f'Status: {session[\"status\"]}')
assert session['status'] == 'created'

# 更新
session['status'] = 'in_progress'
update_session(sid, session)
session2 = get_session(sid)
assert session2['status'] == 'in_progress'

print('Session CRUD 验证通过！')
"
```
