## 学习内容（~2 小时）

### Part 1：面试复习（Doc 03 §6）

练习回答以下 4 个面试问题：

**Q1: 模拟面试的多轮对话上下文是怎么管理的？**
> 要点：Redis 存储面试 session，包含完整 messages 数组。每次用户回答后更新 session，下一次出题/追问时带上全量历史。设置 2 小时 TTL 自动过期。对于超长面试，可以切换到滑动窗口 + 摘要压缩策略。

**Q2: Skill 出题是怎么保证不重复的？**
> 要点：每次出题时把已问过的题目列表传给 LLM，在 prompt 中明确要求不要重复。同时在 session 中维护 `questions_asked` 数组做本地去重。双重去重：prompt 级 + 应用级。

**Q3: 评估引擎为什么分批而不是一次性评估？**
> 要点：一次性评估 10-20 道题的问答 prompt 会很长，LLM 注意力分散，评分质量不稳定。分批每次处理 3 题，LLM 能更专注。分批还可以并发调用加快速度。最后做一次汇总生成总评报告。

**Q4: 面试邀请解析为什么用"规则 + AI 双引擎"？**
> 要点：规则引擎处理标准格式（固定的时间/链接正则模式）非常可靠且零成本。AI 引擎处理非标准文本更灵活。双引擎结合：规则结果优先（可靠），AI 补充缺失字段（灵活）。

### 能讲出的亮点

1. **Skill 驱动出题**：面试方向可配置，通过 Markdown 文件定义考察范围和难度分布，扩展新方向不改代码
2. **评估引擎架构**：分批评估 + 结构化输出 + 二次汇总，质量可控
3. **历史题目去重**：prompt 级 + 应用级双重去重
4. **会话缓存**：Redis + TTL，支持断点续面
5. **双引擎解析**：规则 + AI 互补，兼顾可靠性和灵活性

## 编码任务

### 任务 1：完整面试流程端到端测试

启动服务后，执行完整面试流程：

```bash
# 启动服务
uvicorn app.main:app --reload

# Step 1: 创建面试（5 题的短面试）
curl -X POST http://localhost:8000/interview/start \
  -H "Content-Type: application/json" \
  -d "{\"skill\": \"python_backend\", \"total_questions\": 5}"
# 记录返回的 session_id

# Step 2: 连续回答 5 题
curl -X POST http://localhost:8000/interview/answer \
  -H "Content-Type: application/json" \
  -d "{\"session_id\": \"<ID>\", \"answer\": \"GIL 是全局解释器锁，限制同一时刻只有一个线程执行字节码\"}"

# 重复 Step 2 直到返回 status: completed
# （每次回答可以用不同的内容测试）

# Step 3: 获取评估报告
curl -X POST "http://localhost:8000/interview/evaluate?session_id=<ID>"
# 预期返回：overall_score + summary + strengths + improvements + items
```
