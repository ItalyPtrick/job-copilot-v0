## 学习内容（~2 小时）

### Part 1：面试复习问题

**Q1: 模拟面试的多轮对话上下文是怎么管理的？**
> Redis 存储 session，保存 config、messages、当前主问题、已追问次数、已覆盖考点、已问主问题和最近表现摘要。messages 保留完整历史，assistant 消息用 metadata 标记 main / follow_up。短面试可全量保存，长面试可切换到滑动窗口 + 摘要。

**Q2: Skill 出题为什么要先转成蓝图，而不是每次全文传入 prompt？**
> Markdown Skill 适合作为配置源，但每次全文传入会增加 token 成本并稀释注意力。蓝图化后，每轮只传当前相关的考点、难度 rubric、已覆盖维度和已问列表，既保留可配置性，也让 prompt 更聚焦。

**Q3: 难度是怎么控制的？**
> 先根据 total_questions 和 difficulty_distribution 生成基础难度序列，例如 5 题对应 easy、easy、medium、medium、hard。每道题生成时把目标难度的 rubric 注入 prompt，要求模型返回 difficulty_reason。planner 会结合 recent_performance 对下一题难度做最多一档的动态调整；应用层校验返回 difficulty 必须和请求一致。

**Q4: 追问为什么用 planner，而不是每题固定追问？**
> 真人面试不会机械追问。planner 根据回答质量、追问次数上限和面试进度决定 follow_up / next_question / complete。回答模糊或暴露误区时追问，回答充分时推进下一题，节奏更自然。

**Q5: 评估引擎为什么按主问题轮次评估？**
> 追问是主问题的一部分，不应被当成同等权重的独立题。按 turn 聚合主问题、初答和追问回答，能更准确评估候选人最终掌握程度，也能看出追问后是否补救成功。

**Q6: 面试邀请解析为什么用规则 + AI 双引擎？**
> 规则引擎处理标准时间和会议链接可靠、零成本；AI 引擎处理非标准描述更灵活。合并时规则优先，AI 补充缺失字段。

### 能讲出的亮点

1. **Skill 蓝图化**：Markdown 配置外置，运行时转成聚焦 prompt 的结构化蓝图。
2. **难度 rubric 控制**：easy / medium / hard 不只是标签，而有明确能力要求。
3. **自适应追问 planner**：根据回答质量决定追问、下一题或结束。
4. **主追问聚合评估**：追问不单独计题，而归入主问题轮次评分。
5. **Redis 会话缓存**：TTL + session 状态支持断点续面。
6. **双引擎邀请解析**：规则可靠、AI 灵活。

## 编码任务

### 任务 1：完整模拟面试端到端测试

启动服务后执行：

```bash
uvicorn app.main:app --reload

curl -X POST http://localhost:8000/interview/start \
  -H "Content-Type: application/json" \
  -d "{\"skill\": \"python_backend\", \"total_questions\": 5, \"follow_up_count\": 1}"

curl -X POST http://localhost:8000/interview/answer \
  -H "Content-Type: application/json" \
  -d "{\"session_id\": \"<ID>\", \"answer\": \"GIL 是全局解释器锁，限制同一时刻只有一个线程执行字节码\"}"
```

连续回答，观察：

- 有时返回追问。
- 有时直接返回下一道主问题。
- 主问题进度只统计主问题，不统计追问。
- 达到 total_questions 后返回 completed。

### 任务 2：获取评估报告

```bash
curl -X POST "http://localhost:8000/interview/evaluate?session_id=<ID>"
```

预期返回：overall_score + summary + strengths + improvements + items。

### 任务 3：验证面试安排解析

```bash
curl -X POST http://localhost:8000/schedule/parse-invite \
  -H "Content-Type: application/json" \
  -d "{\"text\": \"您好，邀请您参加字节跳动 AI 应用开发实习面试，时间：2025-05-01 14:00，腾讯会议链接：https://meeting.tencent.com/dm/test123，面试官：张老师\"}"
```

### 任务 4：运行测试和清理

```bash
pytest tests/ -v
```

检查：

- import 无冗余。
- interview / schedule 路由在 Swagger 中展示正常。
- `app/skills/python_backend.md` 内容完整。
- `docs/03-mock-interview.md`、`docs/00-roadmap.md`、W3 日计划已对齐。
