## 学习内容（~30 分钟）

复习 Doc 03 §1 "评估引擎" 的架构和 §4 Step 4 的完整代码，理清 4 个函数的职责：

1. **`_extract_qa_pairs(messages)`**：从对话历史中提取问答对（assistant 的问题 + user 的回答配对）
2. **`evaluate_batch(qa_pairs)`**：分批评估，每次 3-5 题，LLM 返回结构化评分 JSON 数组
3. **`generate_report(all_evaluations)`**：汇总所有单题评分，生成总评报告（overall_score / summary / strengths / improvements）
4. **`evaluate_interview(messages)`**：完整评估流程入口——提取问答对 → 分批评估 → 汇总报告

关键设计：
- **评分标准**内嵌在 prompt 中：1-3 错误、4-5 了解但不深、6-7 正确有深度、8-9 出色有经验、10 完美
- **分批大小**：batch_size=3，平衡精度和调用次数
- **问答对提取逻辑**：assistant 消息含 `?` 视为问题，下一条 user 消息视为回答

## 编码任务

### 任务 1：创建评估引擎（Doc 03 Step 4）

创建 `app/modules/interview/evaluation.py`，参考 Doc 03 Step 4 的代码：

**函数 1：`_extract_qa_pairs(messages)`**
- 输入：`messages` 列表（`[{"role": "assistant", "content": "..."}, {"role": "user", "content": "..."}, ...]`）
- 逻辑：遍历 messages，assistant 消息含 `?` 则记为当前问题，下一条 user 消息配对为回答
- 输出：`[{"question": "...", "answer": "..."}, ...]`

**函数 2：`evaluate_batch(qa_pairs)`**
- 输入：3-5 个问答对
- 构造 system_prompt：评分标准（1-10 分）+ JSON 输出格式
- 调用 `call_llm`，返回评分数组
- 每项包含：question / score / feedback / category

> **⚠️ 静默失败风险**：`call_llm` 类型标注 `-> dict` 但 JSON 数组会实际返回 `list`——这是正常的。真正的问题是：如果 LLM 返回格式异常，`call_llm` 返回 `{"error": ..., "raw": ...}`，`isinstance(result, list)` 为 False，函数默认返回 `[]`。这会导致部分题目的评分**静默丢失**，而不报任何错。建议在 `else` 分支加 `import logging; logging.warning(f"evaluate_batch 解析失败: {result}")` 以便调试。

**函数 3：`generate_report(all_evaluations)`**
- 输入：所有单题评分结果
- 构造 system_prompt：要求生成总评 JSON（overall_score / summary / strengths / improvements）
- 调用 `call_llm`

**函数 4：`evaluate_interview(messages)`** — 流程编排
```python
def evaluate_interview(messages):
    qa_pairs = _extract_qa_pairs(messages)
    batch_size = 3
    all_evals = []
    for i in range(0, len(qa_pairs), batch_size):
        batch = qa_pairs[i:i + batch_size]
        evals = evaluate_batch(batch)
        all_evals.extend(evals)
    report = generate_report(all_evals)
    report["items"] = all_evals
    return report
```

- `from app.services.llm_service import call_llm`
