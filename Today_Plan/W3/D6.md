## 学习内容（~30 分钟）

今天有两块：

1. **面试安排解析**
   - 规则引擎提取标准时间和会议链接。
   - AI 引擎补充公司、岗位、面试官、备注等非标准字段。
   - 合并时规则结果优先。

2. **模拟面试测试补齐**
   - 新版 W3 的复杂点集中在 planner 与主追问消息结构。
   - 测试重点应覆盖“是否追问、何时进入下一题、追问不计入主问题数、评估能聚合主追问”。

## 编码任务

### 任务 1：创建面试邀请解析器

创建 `app/modules/schedule/invite_parser.py`：

- `parse_invite_rule_based(text)`：正则提取时间、腾讯会议 / Zoom / 飞书链接。
- `parse_invite_ai(text)`：调用 LLM 提取 company / position / start_time / end_time / meeting_link / interviewer / notes。
- `parse_invite(text)`：规则优先，AI 补充缺失字段。

### 任务 2：创建 schedule 路由

创建 `app/modules/schedule/router.py`：

- `POST /schedule/parse-invite`

并在 `app/main.py` 注册 `/schedule` 路由。

### 任务 3：补 W3 测试

推荐拆分测试文件。本节列出的 planner / evaluation / invite parser 测试由 D6 当天补齐；在 D5/D6 实现前，不要求这些文件已经存在。

- `tests/test_question_engine.py`
  - Skill blueprint 构建
  - 难度 rubric 注入
  - 重复题拒绝
  - follow_up raw 兜底

- `tests/test_interview_planner.py`
  - 从 `app.modules.interview.interview_planner` 导入 `plan_next_interview_action`
  - 回答模糊时返回 follow_up
  - 追问次数达到上限时返回 next_question
  - 主问题达到总数时返回 complete
  - `total_questions=5` 默认生成 `easy, easy, medium, medium, hard`
  - `performance_signal=strong/weak` 时最多上下浮动一档

- `tests/test_interview_evaluation.py`
  - 主问题 + 追问被提取到同一 turn
  - 追问不被当成独立主问题评分

- `tests/test_schedule_invite_parser.py`
  - 规则解析腾讯会议/飞书/Zoom 链接
  - mock AI 解析结果，验证 `parse_invite` 保留规则提取的时间和会议链接，只用 AI 补充 company / position 等缺失字段
