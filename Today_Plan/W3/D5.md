## 学习内容（~30 分钟）

阅读 Doc 03 §4 Step 5 的完整路由代码和 §3 的集成点说明：

- 3 个路由端点的职责：
  - `POST /interview/start`：创建 session → 加载 Skill → 出第一题 → 返回 session_id + question
  - `POST /interview/answer`：验证 session → 记录回答 → 判断是否结束 → 出下一题或返回完成状态
  - `POST /interview/evaluate`：验证 session 状态 → 调用评估引擎 → 返回结构化报告
- 理解路由层如何串联 session_manager、question_engine、evaluation 三个模块

## 编码任务

### 任务 1：创建面试路由（Doc 03 Step 5）

创建 `app/modules/interview/router.py`，参考 Doc 03 Step 5 的代码：

**请求模型**：
```python
class StartInterviewRequest(BaseModel):
    skill: str = "python_backend"
    total_questions: int = 10

class AnswerRequest(BaseModel):
    session_id: str
    answer: str
```

**路由 1：`POST /interview/start`**
- 调用 `create_session(req.model_dump())` 创建 session
- 用 `open(f"app/skills/{req.skill}.md")` 加载 Skill 文件
- 调用 `generate_question(skill_content, "easy", [])` 生成第一题（第一题固定 easy 难度）
- 更新 session：status → "in_progress"，添加 messages 和 questions_asked
- 返回 `{"session_id": ..., "question": ...}`

**路由 2：`POST /interview/answer`**
- 从 Redis 获取 session，验证存在性
- 将回答追加到 `session["messages"]`
- 判断 `current_question_index >= total_questions` → 返回完成状态
- 否则生成下一题（medium 难度），更新 session
- 返回 `{"question": ..., "progress": "3/10"}`

**路由 3：`POST /interview/evaluate`**
- 验证 session 状态为 "completed"
- 更新状态为 "evaluating"
- 调用 `evaluate_interview(session["messages"])`
- 更新状态为 "evaluated"，存入报告
- 返回评估报告

**路由注册**：`router = APIRouter(prefix="/interview", tags=["模拟面试"])`

### 任务 2：修改 main.py 注册路由

在 `app/main.py` 中添加：

```python
from app.modules.interview.router import router as interview_router
app.include_router(interview_router)
```

### 任务 3：扩展 redis_client.py（可选）

如果需要面试 session 专用的便捷方法，在 `app/cache/redis_client.py` 中添加：

```python
# 面试 session 专用操作
INTERVIEW_PREFIX = "interview:session:"

def set_interview_session(session_id: str, data: dict, ttl: int = 7200) -> None:
    set_session(f"{INTERVIEW_PREFIX}{session_id}", data, ttl)

def get_interview_session(session_id: str) -> dict | None:
    return get_session(f"{INTERVIEW_PREFIX}{session_id}")
```

> 这一步可选——如果 `session_manager.py` 已经直接用 `redis_client.setex` 加了前缀，就不需要额外封装。

### 任务 4：启动服务并手动验证

